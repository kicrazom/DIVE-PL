An AI tool-chain for building survey studies — made reproducible

DIVE-PL specifies and demonstrates an end-to-end pipeline for designing, fielding and analysing questionnaire research using an integrated chain of AI tools, wrapped in an open-science layer that makes every step independently verifiable.

Zenodo DOI GitHub repository Docs: CC BY 4.0 Code: MIT

Human-in-the-loop boundary. Every AI output is reviewed and approved by a domain expert before it affects the study. AI augments; it does not replace expert judgment, and AI tools are not co-authors. See Limitations.

The contribution

The novelty is the integration, not any single technique. Each component has prior art; to our knowledge the combination of all four components plus a full open-science reproducibility layer has not been published as a single survey-study methodology (literature scan, mid-2026). See Related work.

The four components

1 · LLM agent

Claude / Cowork designs & orchestrates the study: protocol, item drafting, registration artifacts, workflow.

2 · LLM reviewer

GPT acts as an independent LLM-as-judge, flagging bias, ambiguity, and content-validity issues.

3 · REDCap

Electronic data capture: validated instrument, consent / RODO logic, de-identified data store.

4 · Dashboards

AI-built Python / React analytics, published openly via GitHub Pages.

Case study

The framework is demonstrated on a cross-sectional online survey of recreational divers in Poland (health, fitness via a non-exercise VO2max proxy, safety knowledge, and self-reported adverse events). The study is ethics-approved and registered (protocol APK.002.228.2026). No participant data are published here.
